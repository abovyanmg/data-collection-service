# 📦 Инструкция по обновлению контейнеров ACS

## Обновление до новой версии с обновленным отчетом nmreport

---

## ⚠️ Важно перед началом

- **Сохраните настройки**: Убедитесь, что файл `settings.xlsx` сохранен и доступен
- **Резервная копия**: Если есть важные данные, сделайте бэкап
- **Время обновления**: Процесс займет 5-10 минут

---

## 📋 Шаг 1: Проверка текущих контейнеров

### 1.1. Посмотреть все запущенные контейнеры

```bash
docker ps -a
```

Вы увидите что-то вроде:
```
CONTAINER ID   IMAGE                              STATUS
abc123def456   abovyanmg/acs-upload-data:stable   Up 2 hours
def456ghi789   abovyanmg/acs-upload-data:stable   Up 1 day
ghi789jkl012   clickhouse/clickhouse-server:stable Up 3 days
```

**Запомните имена контейнеров** с `acs-upload-data` - их нужно будет обновить.

---

## 🛑 Шаг 2: Остановка контейнеров upload_data

### 2.1. Остановить ВСЕ контейнеры upload_data

**Важно:** Нужно остановить ВСЕ контейнеры, которые используют образ `acs-upload-data`, не только основной!

#### Вариант A: Если используете docker-compose

```bash
# Перейдите в папку с docker-compose.yml
cd /path/to/your/project

# Остановить все контейнеры
docker-compose stop
```

#### Вариант B: Если контейнеры запущены вручную

```bash
# Остановить конкретный контейнер (замените имя на ваше)
docker stop upload_data_example

# Или остановить все контейнеры с acs-upload-data
docker ps --filter "ancestor=abovyanmg/acs-upload-data:stable" --format "{{.Names}}" | xargs docker stop
```

#### Вариант C: Остановить все контейнеры upload_data по имени

```bash
# Остановить все контейнеры, в имени которых есть "upload_data"
docker ps --format "{{.Names}}" | grep upload_data | xargs docker stop
```

### 2.2. Проверить, что контейнеры остановлены

```bash
docker ps
```

Контейнеры `upload_data` должны быть в статусе `Exited` или не отображаться в списке запущенных.

---

## 🗑️ Шаг 3: Удаление старых контейнеров

### 3.1. Удалить остановленные контейнеры upload_data

**Важно:** Это удалит только контейнеры, НЕ данные! Настройки и данные ClickHouse останутся.

#### Вариант A: Если используете docker-compose

```bash
# Удалить контейнеры (данные в volumes сохранятся)
docker-compose rm -f
```

#### Вариант B: Удалить контейнеры вручную

```bash
# Удалить конкретный контейнер
docker rm upload_data_example

# Или удалить все остановленные контейнеры с acs-upload-data
docker ps -a --filter "ancestor=abovyanmg/acs-upload-data:stable" --format "{{.Names}}" | xargs docker rm
```

### 3.2. Проверить, что контейнеры удалены

```bash
docker ps -a | grep upload_data
```

Должно быть пусто или показать только контейнеры, которые вы хотите оставить.

---

## 🗑️ Шаг 4: Удаление старых образов

### 4.1. Посмотреть текущие образы

```bash
docker images | grep acs-upload-data
```

Вы увидите что-то вроде:
```
abovyanmg/acs-upload-data   stable   416a6fcc3f90   846MB   2 weeks ago
abovyanmg/acs-upload-data   latest   416a6fcc3f90   846MB   2 weeks ago
```

### 4.2. Удалить старые образы

```bash
# Удалить образ stable
docker rmi abovyanmg/acs-upload-data:stable

# Удалить образ latest (если используется)
docker rmi abovyanmg/acs-upload-data:latest

# Если Docker спрашивает подтверждение - нажмите 'y' или используйте флаг -f
docker rmi -f abovyanmg/acs-upload-data:stable
```

**Примечание:** Если образ используется другими контейнерами, сначала удалите их.

### 4.3. Проверить, что образы удалены

```bash
docker images | grep acs-upload-data
```

Должно быть пусто или показать только образы, которые вы хотите оставить.

---

## 📥 Шаг 5: Скачивание новых образов

### 5.1. Скачать обновленный образ

```bash
# Скачать образ stable (рекомендуется)
docker pull abovyanmg/acs-upload-data:stable

# Или скачать latest
docker pull abovyanmg/acs-upload-data:latest
```

Вы увидите процесс скачивания:
```
stable: Pulling from abovyanmg/acs-upload-data
...
Status: Downloaded newer image for abovyanmg/acs-upload-data:stable
```

### 5.2. Проверить, что образ скачан

```bash
docker images | grep acs-upload-data
```

Должен появиться новый образ с актуальной датой:
```
abovyanmg/acs-upload-data   stable   566950ebfac7   846MB   2 minutes ago
```

---

## 🚀 Шаг 6: Создание и запуск новых контейнеров

### 6.1. Если используете docker-compose

```bash
# Создать и запустить контейнеры
docker-compose up -d

# Проверить статус
docker-compose ps
```

### 6.2. Если контейнеры запускаются вручную

```bash
# Пример команды для создания контейнера
docker run -d \
  --name upload_data_example \
  --network data_network \
  -v /path/to/settings.xlsx:/app/settings.xlsx:ro \
  -e CLICKHOUSE_HOST=clickhouse \
  -e CLICKHOUSE_PORT=8123 \
  abovyanmg/acs-upload-data:stable
```

**Важно:** Замените параметры на ваши:
- `--name` - имя контейнера
- `--network` - имя сети Docker
- `-v` - путь к файлу settings.xlsx
- `-e` - переменные окружения

---

## ✅ Шаг 7: Проверка работы

### 7.1. Проверить статус контейнеров

```bash
docker ps
```

Все контейнеры должны быть в статусе `Up`.

### 7.2. Проверить логи

```bash
# Логи конкретного контейнера
docker logs upload_data_example

# Или если используете docker-compose
docker-compose logs upload_data_example

# Логи в реальном времени
docker logs -f upload_data_example
```

### 7.3. Проверить, что nmreport работает

В логах должны быть сообщения о сборе данных:
```
Платформа: WB. Имя: ... Дата: ... Функция: get_nmreport. Результат: ОК
```

Если видите ошибки - проверьте настройки в `settings.xlsx`.

---

## 🔄 Альтернативный способ (быстрый)

Если используете `docker-compose`, можно обновить одной командой:

```bash
# Остановить, удалить, скачать новое, запустить
docker-compose down
docker-compose pull
docker-compose up -d
```

## ❓ Частые вопросы

### Нужно ли останавливать ClickHouse?

**Нет!** ClickHouse обновлять не нужно, он использует стандартный образ. Останавливайте только контейнеры `upload_data`.

### Что делать, если контейнер не запускается?

1. Проверьте логи: `docker logs <имя_контейнера>`
2. Проверьте, что файл `settings.xlsx` доступен
3. Проверьте, что ClickHouse запущен: `docker ps | grep clickhouse`
4. Проверьте сеть Docker: `docker network ls`

### Нужно ли обновлять все контейнеры upload_data?

**Да!** Все контейнеры, которые используют образ `acs-upload-data`, нужно обновить, чтобы получить новую версию с обновленным nmreport.

### Что делать с данными в ClickHouse?

**Ничего!** Данные в ClickHouse сохраняются в volumes и не удаляются при обновлении контейнеров.
